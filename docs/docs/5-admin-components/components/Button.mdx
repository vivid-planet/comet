## Responsive buttons

To save space, e.g. inside a toolbar on mobile devices, you can set the `responsive` prop. <br/>
This will only show the icon by default and the text content will be added as a tooltip around the `Button`.

An icon is required for the responsive behavior to work, use either the `startIcon`, `endIcon` or `mobileIcon` prop.

_Make your browser window smaller to see the responsive behavior._

## The `FeedbackButton` component

Use the `FeedbackButton` for buttons that should give feedback to the user, after clicked. <br/>
Simply pass in an async function to `onClick` will show a loading state, followed by a success or error tooltip.

An icon is required for the feedback tooltip to be shown, use either `startIcon` or `endIcon`.

The messages displayed in the feedback can be customized using the `tooltipSuccessMessage` and `tooltipErrorMessage` props.
